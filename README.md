# Day 5 – SIEM (Security Information and Event Management)

## Overview
This repository contains my Day 5 learning on SIEM, focusing on its architecture, workflow, and role in Security Operations Center (SOC) environments.

---

## What is SIEM?
SIEM (Security Information and Event Management) is a centralized system used to collect, analyze, and correlate logs from multiple sources to detect and respond to security threats in real time.

---

## Key Concepts
- Log collection from servers, firewalls, endpoints, and network devices  
- Log normalization and storage for analysis  
- Correlation of events to detect suspicious patterns  
- Alert generation for potential threats  
- Visualization using dashboards  

---

## SIEM Workflow
1. Log Collection  
2. Log Processing (Normalization & Storage)  
3. Correlation  
4. Threat Detection  
5. Alert Generation  

---

## Popular SIEM Tools
- Splunk  
- ELK Stack (Elasticsearch, Logstash, Kibana)  
- IBM QRadar  

---

## SOC Use Cases
- Detecting brute force attacks  
- Identifying malware activity  
- Monitoring insider threats  
- Investigating security incidents  
- Real-time threat detection and response
